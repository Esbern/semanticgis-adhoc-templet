# Analytical Recipe

Describe analysis logic independent of execution engine.

## Design Brief Template

For each multi-step analysis, fill in a design brief below **before** writing code.
Present it to the user for approval. Simple single-operation tasks may skip this.

### Brief: [Analysis Name]

- **Objective**: What question does this analysis answer?
- **Input data**: Which datasets (from `00_Binary/derived/`) are consumed?
- **Method**: What operations are performed, in what order? (plain language, no code)
- **Assumptions**: What spatial, temporal, or methodological assumptions are made?
- **Expected output**: What is produced? (file format, location, key columns/layers)
- **Approval**: [ ] User approved / [ ] Revision requested

---

_After implementation, note any deviations from the approved brief below the relevant entry._
